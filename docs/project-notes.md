# TurboCodAI — заметки проекта

Автор: Sementsul Maxim. (Папка репозитория исторически `deepseek-claude`.)


Одностраничный браузерный AI-клиент (OpenAI-совместимый) + опциональный бэкенд-прокси.

## Состав
- `index.html` — самодостаточный фронт. Возможности:
  - Чат со стримингом, тёмная/светлая тема, выбор провайдера.
  - **Профили ИИ** (левая панель): несколько подключений (провайдер/URL/модель/ключ), переключение.
    Тема и системный промпт — общие. Кнопка «💾 Сохранить настройки».
  - **Мультичаты**: создание/удаление/переключение; история в localStorage (`ds_chats`),
    указатель активного чата — в куках+localStorage. (Куки ~4 КБ — всю историю не вмещают, поэтому
    она в localStorage; в куках только указатели.)
  - **Загрузка файлов** в чат (📎): текстовые инлайнятся в сообщение; картинки → data URL (vision).
  - **Наборы промптов**: встроенные (35) + файлы из `prompts/` (манифест `prompts/index.json`).
    Два выпадающих списка: набор → пресет. На file:// — только встроенные.
  - **Artifacts**: пресет «Artifacts (генератор кода)» → ИИ отвечает JSON `{response, artifacts[], suggestions[]}`.
    Панель справа: вкладки файлов, живое превью HTML/SVG (sandbox iframe), редактор, экспорт в ZIP
    (свой ZIP-упаковщик на чистом JS, без зависимостей).
  - **Функции (tool-calls)**: тумблер «🔧 Функции» — клиент сам вызывает CORS-дружелюбные API
    (get_weather через open-meteo, get_time, fetch_url). Без стриминга, нужна модель с tool-calls.
  - **Динамический системный промпт** (buildSystemPrompt): пересобирается перед каждым запросом —
    базовый промпт + адаптация под модель (getModelConfig) + резюме истории + список файлов ФС + правила.
  - **Self-refine** («✨ Улучшить ответ»): цикл до 3 итераций — критик-редактор улучшает последний ответ.
  - **Управление контекстом**: авто-сжатие истории при > 24 сообщений (хранит последние 10, остальное →
    резюме через модель в `chat.summary`). Ручное — кнопка в панели контекста.
  - **Виртуальная ФС (IndexedDB)**: read/write/list/search/delete. Модель выводит теги
    `<read_file>`/`<write_file>`/`<grep>`, КЛИЕНТ их выполняет и возвращает результаты в контекст (агентный цикл, до 3 шагов).
  - **Панель контекста** (🧠): счётчики сообщений/сжатия, последние 5 сообщений, файлы ФС (с удалением),
    кнопки «Сжать контекст» / «Очистить историю».
  - **Готовые профили** (➕): DeepSeek, OpenRouter, Ollama (localhost:11434), Google Gemini.
  - **Экспорт/импорт сессии** (⬇/⬆): JSON с messages + файлами ФС + настройками/профилями.
  - **Команда агентов** (🤖, режим в основном чате): тумблер 🤖 у поля ввода (`agentMode`). Когда включён —
    на каждое сообщение координатор сам решает (`coordDecideSys` → `needsTeam`): если задача требует НЕСКОЛЬКО
    файлов/слоёв → подключает до N агентов (панель 🤖 Команда задаёт макс. N=2..4); если простая — отвечает один.
    Агенты пишут код ПАРАЛЛЕЛЬНО на ОДНОМ ключе (Promise.all, тот же Authorization). Координатор проверяет
    согласованность (имена/интерфейсы/дубли), исправляет и собирает финальный набор → панель Артефактов + ZIP.
    **Стрим-лог**: весь прогресс идёт прямо в ленту чата — `streamCall` льёт токены каждого агента/координатора
    в своё сообщение (моноширинный лог), затем заменяется на сводку (`✅ Агент N: файлы …`).
    **Цикл доработки** (до 3 кругов): `done:false` + issues → агенты правят свою зону (currentFiles + issues) →
    перепроверка; стоп при `done:true`/исчерпании. mergeFiles (правки приоритетнее базы), pickFinal.
  - **Мультипрофильная команда:** координатор работает на АКТИВНОМ профиле; worker-агенты можно раскидать по
    другим профилям (чекбоксы в панели 🤖, `ds_team`). streamCall принимает prof → epOf/hdrOf/mdlOf (свой
    base/ключ/модель на агента, round-robin team[i%len]). Разные ключи = реальная параллельность (обход лимита
    одного ключа) + микс моделей. Если профили не отмечены — все на активном. В логе видно [профиль] и тайминги
    (старт/1-й токен/готово) — по «1-му токену» видно, идут параллельно или сериализуются.
  - **База знаний** (кнопка «База знаний» в шапке): панель с описанием всех возможностей клиента (константа `KB`).
    Системный промпт `KB_SYS` (вступление + KB) доступен как пресет «О TurboCodAI (база знаний)» и по кнопке
    «Спросить ИИ о клиенте» — тогда ИИ точно отвечает о самом клиенте. ⚠️ ПРАВИЛО: при добавлении нового
    функционала ОБНОВЛЯТЬ константу `KB` в index.html (она же идёт в KB_SYS и в панель).
  - Иконки интерфейса — инлайн-SVG (стиль Lucide), без CDN (`ICON`/`applyIcons`). Бренд: TurboCodAI, автор Sementsul Maxim.
  - Подсветка синтаксиса без CDN: `highlight()` (универсальная), `highlightHtml()`, `highlightCss()` — в код-блоках чата и редакторе артефактов (оверлей textarea+pre).
  - **Локализация** RU/EN (`I18N`/`t()`/`applyLang()`, кнопка-язык в шапке, `ds_lang`, по умолчанию RU). Перевод через `data-i18n`/`data-i18n-ph` + подписи кнопок через t(). KB остаётся RU (ИИ отвечает на языке пользователя).
  - Число агентов команды — любое (поле number в панели, было 2–4).
    Произвольные URL из браузера ограничены CORS — общий случай решает бэкенд `/tools/chat`.
  - Хранилище: куки + localStorage. На `file://` ненадёжно — запускать через http-сервер / GitHub Pages.
- `prompts/` — наборы промптов: `index.json` (манифест), `claude-code-models.json` (выбор «модели Claude Code»),
  `roles.json` (роли разработки). `prompts.json` в корне — legacy-набор (тоже подхватывается).
- `backend/` — FastAPI-прокси (см. backend/README.md).

## Бэкенд (backend/)
- `POST /v1/chat/completions` — OpenAI-совместимый прокси со стримингом. Прячет ключ LLM.
- `POST /tools/chat` — Function Calling: инструменты get_weather (open-meteo), fetch_data (SSRF-защита),
  search_web (DuckDuckGo IA). Бэкенд сам гоняет цикл tool_calls.
- `GET /health`.
- Безопасность: токен-гейт CLIENT_TOKENS, CORS по ALLOWED_ORIGINS, rate-limit на IP, SSRF-guard.
  Реальный ключ LLM — только в .env (в .gitignore). **Секреты — красная зона, под ревью ПМ.**
- Тесты: `backend/test_app.py` (чистая логика, без сети) — 5/5 зелёные.

## Архитектура для GitHub Pages
GitHub Pages = только статика. Фронт хостится на github.io, бэкенд — отдельно (Render/Railway/Fly).
Фронт ставит Base URL = `https://<бэкенд>/v1`, в «API-ключ» — клиентский токен (CLIENT_TOKENS).

## Как запускать локально
- Фронт: `python3 -m http.server 8000 --bind 0.0.0.0` → http://localhost:8000
- Бэк: `cd backend && uvicorn app:app --host 0.0.0.0 --port <свободный>` (нужен .env с LLM_API_KEY)

## Грабли
- На `file://` куки не сохраняются, localStorage часто тоже → настройки слетают. Решение: http-сервер/Pages.
- OpenAI блокирует CORS из браузера; DeepSeek/OpenRouter/Groq — нет. Через бэкенд-прокси CORS не проблема.
- Публичный прокси БЕЗ токен-гейта = любой сольёт твой платный баланс. CLIENT_TOKENS обязателен в проде.
