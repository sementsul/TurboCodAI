# TurboCodAI — браузерный AI-клиент для GitHub Pages

**Автор:** Sementsul Maxim


Одностраничный браузерный AI-клиент: чат со стримингом, профили ИИ, мультичаты,
загрузка файлов, наборы промптов, сборка проекта из файлов в ZIP, тёмная/светлая тема.
Всё — статика, работает на GitHub Pages. Ничего на сервере не хранится.

## 🚀 Развернуть на GitHub Pages (2 шага)

1. Создай репозиторий и положи в его **корень** содержимое этого архива
   (важно: `index.html` должен быть в корне, не во вложенной папке).
2. GitHub → **Settings → Pages → Build and deployment → Source:**
   - **«Deploy from a branch»** → ветка `main`, папка `/ (root)` → **Save**.
     Через минуту сайт будет на `https://USERNAME.github.io/REPO/`.
   - *или* **«GitHub Actions»** — тогда задеплоит автоматически по push
     (workflow уже лежит в `.github/workflows/deploy-pages.yml`).

Открой выданный адрес → ⚙ Настройки → впиши свой **API-ключ** (DeepSeek/OpenRouter/…) → пиши.

> 🔑 Ключ вводится в браузере и хранится только у тебя (localStorage + куки).
> В репозиторий он НЕ попадает — поэтому коммитить ключ не нужно и нельзя.

## Что внутри
- `index.html` — весь клиент (один файл, без зависимостей).
- `prompts/` — наборы промптов (`index.json` — список; на Pages подхватываются).
- `prompts.json` — legacy-набор (35 пресетов).
- `.nojekyll` — отключает Jekyll, чтобы Pages отдавал файлы как есть.
- `backend/` — **опционально.** FastAPI-прокси, который прячет ключ LLM и добавляет
  Function Calling. На GitHub Pages НЕ запускается — хостится отдельно (Render/Railway).
  См. `backend/README.md`. Для простого использования бэкенд не нужен.

## Провайдеры (CORS из браузера)
- Работают напрямую: **DeepSeek, OpenRouter, Groq** и свой бэкенд-прокси.
- **OpenAI** блокирует запросы из браузера (CORS) — для него нужен бэкенд-прокси.
